Zero-Base LLM

A fully custom language model built from the ground up — starting from raw binary bits (0/1) all the way to paragraph generation, with no borrowed pre-trained weights.

22-Layer Architecture — 6 Zones

Zone A · Layers 0–3
Binary → Byte → ASCII foundation
Zone B · Layers 4–8
Multi-head transformer core (GELU)
Zone C · Layers 9–12
Character clustering → word vectors
Zone D · Layers 13–18
Sentence & paragraph assembly
Zone E · Layers 19–20
Output projection & nucleus sampling
Zone F · Layers 21–22
Self-study: forward + backward consistency

Optimizations Applied

  • GELU activation (better than ReLU for language modeling)
  • GPT-style scaled weight initialization
  • Cosine LR schedule with linear warmup
  • Label smoothing (0.1) for better generalization
  • Nucleus sampling with repetition penalty
  • Reduced dropout (0.1) — standard transformer value

Benchmarks (same metric types as GPT/Claude research)

  • BPC — Bits Per Character (lower = better language model)
  • Perplexity — How "surprised" the model is by text (lower = better)
  • Top-1 / Top-5 Accuracy — Next character prediction quality
  • Diversity (TTR) — Type-token ratio of generated text
  • Repetition Rate — N-gram repetition detection
  • Fluency Score — Real word formation quality

Honest Comparison with Claude Opus

Claude Opus has ~100–200 billion parameters, trained on trillions of tokens. This model has ~10 million parameters trained on hundreds of sentences. They use entirely different benchmarks (MMLU, HumanEval, etc.). The goal here is to make this model as good as it can possibly be within its scale — and to use the same types of scientific measurements used in frontier AI research.
